# BackJumpBF
前方ジャンプ機能がないBrainFuckの実行ハード

## 外部仕様

### 機能一覧

| 項目 | 内容 |
| :--- | :--- |
| BackJumpBF実行機能 | brain fuck("["のジャンプ機能なし)のプログラムを実行することができます<br>プログラム／入力データはシリアル通信からインポートし、システムの状態は7セグLEDもしくはシリアル通信により取得することができます |
| コマンド処理 | システムの設定を確認／更新することができます |
| クロック周波数変更機能 | システムに提供するクロック周波数をハードウェアから制御することができます<br>設定可能周波数は 1Hz から 100Hz です |
| リセット機能 | システムの状態をリセットし、プログラムを削除から実行することができます |

### シリアルデータ入力

- ソフトウェアFIFOに格納されているデータを1文字取得する。
- 取得した文字が何であれ、シリアルデータコールバックタスクの実行フラグを立てる。
- 取得した文字がエンターキー(\r)の場合、シリアルデータ応答タスクの実行フラグを立てる。

### シリアルデータコールバック

- シリアルデータ入力タスクが取得した1文字をコールバックする。

### シリアルデータ応答

- シリアルデータをPCに送信する。

### 構文解析

- シリアルデータ入力タスクが取得した1行(\rまで)の文字列を構文解析する。
- 文字列をトークンに区切り、第一トークンをコマンド種別として扱う。
- 有効なコマンド種別の場合、コマンド実行タスクの実行フラグを立てる。
- 有効なコマンド種別でない場合、異常メッセージを応答するシリアルデータ応答タスクの実行フラグを立てる。

### コマンド実行

- 構文解析タスクが解析したコマンドパラメータを参考にコマンドを実行する。
- 正常にコマンドが完了した場合、コマンド結果を応答するためシリアルデータ応答タスクの実行フラグを立てる。
- 正常のコマンドが完了しなかった場合、異常メッセージを応答するシリアルデータ応答タスクの実行フラグを立てる。

- コマンド一覧
    1. helpコマンド ソフトウェアがサポートしているコマンドの一覧を表示する
    1. setpコマンド コマンド内容をプログラムに反映する
    1. setiコマンド コマンド内容を入力キューに反映する
    1. saveコマンド 現在のプログラム状態をSRAMに保存する
    1. loadコマンド SRAMのプログラム状態をメモリに展開する
    1. fmtコマンド 出力データのシリアル通信フォーマットを反映する

### 入力キュー初期化

- setiコマンドによって設定した入力キューのアクティブアドレスを先頭にリセットする。

### 出力状態更新

- 入力状態更新タスクが管理しているデータをもとに、次の出力状態へ更新する。
- タスクが完了した場合、LED更新タスクの実行フラグを立てる。
- 出力データがある場合、シリアルデータ応答タスクの実行フラグを立てる。

### 入力状態更新

- 出力状態更新タスクか出力しているデータをハード部分の入力とし、ハード部分の出力をマイコンの入力状態とし更新する。

### LED更新タスク

- LEDの出力するIOエキスパンダへ信号を送る。

## 割り込み一覧

- 以下に割り込み一覧を示す。

| No | 概要 |
| :--- | :--- |
| 1 | シリアルデータ1バイト受信 |
| 2 | リセット信号立ち上がり |
| 3 | クロック信号立ち上がり |
| 4 | クロック信号立ち下がり |
| 5 | 10ms定周期割り込み |

### シリアルデータ1バイト受信

- シリアルデータを1バイト受信すると実行される割り込み処理。ソフトウェアFIFOに1バイトのデータを格納する。
- 割り込み処理完了時、シリアルデータ入力タスクの実行フラグを立てる。

### リセット信号立ち上がり

- リセット信号の立ち上がりをすると実行される割り込み処理。
- 割り込み処理完了時、入力キューの初期化タスクの実行フラグを立てる。

### クロック信号立ち上がり

- クロック信号の立ち上がりをすると実行される割り込み処理。
- 割り込み処理完了時、出力状態更新タスクの実行フラグを立てる。

### クロック信号立ち下がり

- クロック信号の立ち下がりをすると実行される割り込み処理。
- 割り込み処理完了後、入力状態更新タスクの実行フラグを立てる。
