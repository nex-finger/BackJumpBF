# BackJumpBF
前方ジャンプ機能がないBrainFuckの実行ハード


## 機能一覧

| 項目 | 内容 |
| :--- | :--- |
| BackJumpBF実行機能 | brain fuck("["のジャンプ機能なし)のプログラムを実行することができます<br>プログラム／入力データはシリアル通信からインポートし、システムの状態は7セグLEDもしくはシリアル通信により取得することができます |
| コマンド処理 | システムの設定を確認／更新することができます |
| クロック周波数変更機能 | システムに提供するクロック周波数をハードウェアから制御することができます<br>設定可能周波数は 1Hz から 100Hz です |
| リセット機能 | システムの状態をリセットし、プログラムを削除から実行することができます |

## 実行までの流れ

1. コントローラのシリアルポート(RS-232C)をデバイスと接続し、通信を開始します
    - ボーレート: 9600bps
    - データ長: 8bit
    - パリティ: なし
    - ストップビット: 1bit

1. コントローラに電源を供給します
    - 電圧: 4.5V ~ 5.5V
    - 消費電流: < 500mA

1. シリアル通信によりシステムが起動していることを確認します  
システムは以下の処理を起動時に自動実行します
    - 内部初期化処理
    - システム名称、ライセンスの表示
    - loadコマンド

1. システムの状態を確認／変更するためコマンドを実行します
    - コマンドはシリアル通信によりシステムに情報を送り、エンターキーで構文解析を開始します
    - 詳細は コマンド一覧 より

1. runコマンドの実行によりBackJumpBFプログラムの実行準備を完了します
    - コマンド実行直後は意味のない処理を繰り返します、後述のリセットボタンにより意味のある処理が開始します
    - runコマンドを実行すると

1. コントローラの CLOCKノブ を調整し、LEDの点滅状態を確認したがら動作させたい周波数に設定します
    - 同じ CLOCKノブ の回転位置の場合でも、供給電圧により周波数は多少変動します

1. コントローラの RESETボタン を押下し、BackJumpBFのプログラム処理が開始されます
    - プログラム実行中に CLOCKノブ を操作することもできます

1. プログラムの停止を検知し次第

## コマンド仕様

- 以下にコマンド一覧を示す。

| No | コマンド名 | 概要 |
| :--: | :--- | :--- |
| 1 | setコマンド | RAMメモリ上のパラメータを更新する |
| 2 | printコマンド | RAMメモリ上のパラメータを表示する |
| 3 | saveコマンド | RAM上のパラメータをEEPROMへ保存する |
| 4 | loadコマンド | EEPROMのパラメータをRAMへ展開する |
| 5 | runコマンド | BackJumpBFプログラムを実行する |
| 6 | helpコマンド | システムの仕様方法を表示する |

### setコマンド

- RAM上のパラメータを更新します

コマンド例
```sh
# プログラムメモリの更新
> set code ?>[-]<[->+<]>![]

# 入力キューの更新
> set queue 0x12 'a' 81

# 出力フォーマットの更新
> set format x(a)
```

#### プログラムメモリの更新

- 第一オペランド: code
- 第二オペランド: コード内容
    - 上限256文字
    - "+-><?![]" のいずれか

#### 入力キューの更新

- 第一オペランド: queue
- 第二オペランド以降: 入力キューひとつずつ
    - d: 10進数(74)
    - a: ascii文字a(J)
    - A: ascii文字b('J')
    - x: 16進数a(4a)
    - X: 16進数b(0x4a)
    - s: スペース
    - t: タブ
    - (, ), -, :, : 即値の文字
    - r: 行頭に移動
    - n: 次の行へ移動

### printコマンド

- RAM上のパラメータを表示します

コマンド例

```sh
# パラメータの表示

```


### helpコマンド


- オペランドなし

## タスク仕様

- 以下にタスク一覧を示す。タスク優先度の数は大きいほど優先度が高い。

| タスクID | 優先度 | 概要 |
| :--: | :--: | :--- |
| -  | high | (↑優先度高) | 
| 7  | 1    | 出力状態更新 |
| 8  | 2    | 入力状態更新 |
| 6  | 3    | 入力キュー初期化 |
| 2  | 7    | シリアルデータコールバック |
| 1  | 4    | シリアルデータ入力 |
| 9  | 5    | LED表示更新 |
| 10 | 6    | 10ms定周期 |
| 3  | 8    | シリアルデータ標準出力 |
| 3  | 9    | シリアルデータエラー出力 |
| 4  | 10   | 構文解析 |
| 5  | 11   | コマンド実行 |
| 11 | 12   | アイドルタスク |
| -  | low  | (↓優先度低)

### シリアルデータ入力

- ソフトウェアFIFOに格納されているデータを1文字取得する。
- 取得した文字が何であれ、シリアルデータコールバックタスクの実行フラグを立てる。
- 取得した文字がエンターキー(\r)の場合、シリアルデータ応答タスクの実行フラグを立てる。

### シリアルデータコールバック

- シリアルデータ入力タスクが取得した1文字をコールバックする。

### シリアルデータ応答

- シリアルデータをPCに送信する。

### 構文解析

- シリアルデータ入力タスクが取得した1行(\rまで)の文字列を構文解析する。
- 文字列をトークンに区切り、第一トークンをコマンド種別として扱う。
- 有効なコマンド種別の場合、コマンド実行タスクの実行フラグを立てる。
- 有効なコマンド種別でない場合、異常メッセージを応答するシリアルデータ応答タスクの実行フラグを立てる。

### コマンド実行

- 構文解析タスクが解析したコマンドパラメータを参考にコマンドを実行する。
- 正常にコマンドが完了した場合、コマンド結果を応答するためシリアルデータ応答タスクの実行フラグを立てる。
- 正常のコマンドが完了しなかった場合、異常メッセージを応答するシリアルデータ応答タスクの実行フラグを立てる。

- コマンド一覧
    1. helpコマンド ソフトウェアがサポートしているコマンドの一覧を表示する
    1. setコマンド コマンド内容をメモリに反映する
    1. printコマンド 現在のメモリ状態を表示する
    1. saveコマンド 現在のプログラム状態をEEPROMに保存する
    1. loadコマンド EEPROMのプログラム状態をメモリに展開する
    1. runコマンド BackJumpBFを実行する

### 入力キュー初期化

- setiコマンドによって設定した入力キューのアクティブアドレスを先頭にリセットする。

### 出力状態更新

- 入力状態更新タスクが管理しているデータをもとに、次の出力状態へ更新する。
- タスクが完了した場合、LED更新タスクの実行フラグを立てる。
- 出力データがある場合、シリアルデータ応答タスクの実行フラグを立てる。

### 入力状態更新

- 出力状態更新タスクか出力しているデータをハード部分の入力とし、ハード部分の出力をマイコンの入力状態とし更新する。

### LED更新タスク

- LEDの出力するIOエキスパンダへ信号を送る。

### アイドルタスク

- NOP命令を1ms続ける

## 割り込み仕様

- 以下に割り込み一覧を示す。

| No | 概要 |
| :--: | :--- |
| 1 | シリアルデータ1バイト受信 |
| 2 | リセット信号立ち上がり |
| 3 | クロック信号立ち上がり |
| 4 | クロック信号立ち下がり |
| 5 | 10ms定周期割り込み |

### シリアルデータ1バイト受信

- シリアルデータを1バイト受信すると実行される割り込み処理。ソフトウェアFIFOに1バイトのデータを格納する。
- 割り込み処理完了時、シリアルデータ入力タスクの実行フラグを立てる。

### リセット信号立ち上がり

- リセット信号の立ち上がりをすると実行される割り込み処理。
- 割り込み処理完了時、入力キューの初期化タスクの実行フラグを立てる。

### クロック信号立ち上がり

- クロック信号の立ち上がりをすると実行される割り込み処理。
- 割り込み処理完了時、出力状態更新タスクの実行フラグを立てる。

### クロック信号立ち下がり

- クロック信号の立ち下がりをすると実行される割り込み処理。
- 割り込み処理完了後、入力状態更新タスクの実行フラグを立てる。
